## Section 1 : Script de Démonstration

### 🎣 Accroche 30 secondes (visiteur qui passe)

> **"AgroGina est la première plateforme ERP + IA décisionnelle conçue pour les fermes marocaines. On voit vos parcelles depuis l'espace — et on vous dit exactement quoi faire, en temps réel."**

*Montrer l'écran principal immédiatement. Ne pas attendre que le visiteur pose des questions.*

---

### 🎬 Parcours démonstration 2 minutes

| Étape | Action | Ce qu'on dit |
|-------|--------|--------------|
| **1. Dashboard** | Ouvrir le tableau de bord — Domaine Toubkal (320 ha, 3 fermes) | *"Voici une exploitation comme la vôtre — Ferme Atlas, Ferme Ziz, Ferme Rif. Tout centralisé, en un coup d'œil."* |
| **2. Parcelles** | Naviguer vers une parcelle d'oliviers | *"Je sélectionne une parcelle d'oliviers. On voit sa superficie, son historique, ses analyses de sol."* |
| **3. Recommandations AgromindIA** | Ouvrir l'onglet recommandations | *"AgromindIA analyse vos données — météo, stocks, main-d'œuvre, santé du sol — et vous donne un conseil actionnable : 'Arrosez la parcelle B3 demain avant 8h.'"* |
| **4. Résultats satellite / calibration** | Afficher le score de santé NDVI + anomalies détectées | *"On détecte le stress hydrique avant que la plante ne le montre. Vous intervenez avant la perte."* |
| **5. Vue satellite** | Ouvrir la carte NDVI colorée sur la parcelle | *"Ça c'est votre ferme vue de l'espace. Chaque couleur = un niveau de santé. En temps réel."* |

---

### 🤝 Clôture

> **"On peut planifier un rendez-vous cette semaine ? Scannez le QR code ou remplissez le formulaire sur rdv.agrogina.ma/rdv-siam — c'est 5 minutes et c'est gratuit."**

*Pointer vers le stand QR code. Remettre le flyer.*

---

### 💡 Adaptations par profil visiteur

| Profil | Ajustement du discours |
|--------|----------------------|
| **Agriculteur (50–300 ha)** | Insister sur AgromindIA niveau 1 — langage simple, bénéfices concrets ("économisez 20% d'eau") |
| **Coopérative** | Montrer la gestion multi-fermes, les exports, la consolidation des données |
| **Distributeur / Partenaire** | Aller directement au modèle économique — SaaS, commissions, territoire exclusif |
| **Investisseur** | Métriques de traction, taille de marché (1,5 M fermes, 9 M ha), différenciateur AgromindIA |
| **Presse / Institution** | Axer sur l'innovation marocaine, la souveraineté numérique agricole, les chiffres d'impact |

## Section 2 : Rôles de l'Équipe

> **Règle d'or** : chaque visiteur qui s'approche doit être accueilli en moins de 10 secondes.

### 👤 Démonstrateur principal

**Mission** : Conduire la démonstration live sur l'écran principal.

- Maîtrise le script de démo (5 étapes, 2 minutes)
- Répond aux questions techniques (satellite, IA, fonctionnalités)
- Ne quitte jamais l'écran principal sans passer la main
- Adapte le discours au profil du visiteur

---

### 👋 Accueillant / Qualificateur

**Mission** : Engager les passants, qualifier les leads, orienter vers la démo.

- Accroche les visiteurs dans l'allée avec la phrase de 30 secondes
- Pose 2–3 questions clés de qualification :
  - *"Vous êtes agriculteur, coopérative ou distributeur ?"*
  - *"Quelle superficie vous gérez ?"*
  - *"Vous utilisez quoi comme outil aujourd'hui ?"*
- Oriente vers le démonstrateur si lead qualifié
- Donne le flyer + pointe vers le QR code si le visiteur est pressé

---

### 📝 Collecteur de contacts / Suivi

**Mission** : Capturer toutes les coordonnées, noter les intérêts, ne laisser passer aucun lead.

- Ouvre le formulaire HubSpot sur tablette pour chaque visiteur
- Champs obligatoires à remplir : Nom, Téléphone WhatsApp, Email, Profil, Superficie, Cultures
- Note dans HubSpot (champ "Notes") : ce qui les a le plus intéressé, questions posées, niveau d'urgence
- Tague les leads chauds pour relance le jour même
- Envoie la séquence email J+0 avant 20h chaque soir

---

### 🔄 Polyvalent (Floater)

**Mission** : Couvrir les pauses, gérer la logistique, ne laisser personne sans réponse.

- Prend le relais du démonstrateur ou de l'accueillant toutes les 2h
- Gère le stock de flyers et cartes de visite
- Publie 1 photo/vidéo sur LinkedIn ou Instagram par demi-journée
- Surveille les appareils (batterie, connexion)
- Interlocuteur pour la logistique salon (badge, colis, traiteur)

---

### 🔁 Rotation recommandée

| Créneau | Démo principal | Accueillant | Collecteur | Floater |
|---------|---------------|-------------|------------|---------|
| 09h–11h | Personne A | Personne B | Personne C | Personne D |
| 11h–13h | Personne B | Personne C | Personne D | Personne A |
| 13h–15h | Personne C | Personne D | Personne A | Personne B |
| 15h–17h | Personne D | Personne A | Personne B | Personne C |
| 17h–19h | Personne A | Personne B | Personne C | Personne D |

*Adapter selon l'effectif réel. Avec 2 personnes : démo + accueil/collecte, rotation à la pause déjeuner.*

## Section 5 : Procédures de Secours

> **Règle d'or** : La démo ne s'arrête jamais. Un des scénarios ci-dessous fonctionne toujours.

---

### 🌐 Scénario 1 — WiFi du salon indisponible

**Symptôme** : L'application ne charge pas les données, les cartes ne s'affichent pas.

**Action** :
1. Allumer le hotspot 4G → connecter les laptops
2. L'application fonctionne normalement via 4G
3. Si la 4G est lente : l'appli charge depuis le cache TanStack Query (données de la session) — continuer la démo normalement
4. Dire au visiteur : *"On est en mode hors-ligne — ça marche exactement pareil, c'est conçu pour ça."*

**Temps de récupération** : < 1 minute

---

### 💥 Scénario 2 — L'application crash sur le laptop principal

**Symptôme** : Crash, écran blanc, bug bloquant.

**Action** :
1. Le 2ème écran continue de diffuser la vidéo en boucle — le visiteur voit toujours quelque chose
2. Ouvrir l'application sur le **laptop backup**
3. Se connecter avec `demo@agrogina.ma` — les données sont identiques
4. Reprendre la démo là où elle s'est arrêtée
5. Redémarrer le laptop principal en parallèle (en fond)

**Temps de récupération** : < 2 minutes

---

### 💀 Scénario 3 — Les 2 laptops sont hors service

**Symptôme** : Panne matérielle grave, écran cassé, laptop mort.

**Action** :
1. La vidéo continue sur le 2ème écran si celui-ci est alimenté séparément
2. Passer en mode **démo papier** :
   - Sortir les screenshots imprimés (préparés à l'avance dans un classeur)
   - Expliquer le produit sur les flyers et le one-pager
3. Collecter les contacts sur le **formulaire papier**
4. Saisir les contacts dans HubSpot depuis le téléphone (app mobile)
5. Dire au visiteur : *"On note vos coordonnées et on vous envoie une démo personnalisée demain."*

**Préparation** : Imprimer 10 pages de screenshots (dashboard, carte NDVI, recommandations AgromindIA) avant le SIAM.

---

### 📡 Scénario 4 — L'API backend est inaccessible (serveur down)

**Symptôme** : L'appli ouvre mais certaines données ne se chargent pas, erreurs 503/504.

**Action** :
1. L'application utilise automatiquement le cache TanStack Query (staleTime 5 minutes + service worker précaché)
2. Toutes les données déjà chargées en session restent visibles
3. **Ne pas recharger la page** — garder les onglets ouverts
4. Si besoin de recharger : le service worker sert le shell de l'appli, les données reviennent du cache
5. Passer à la navigation des sections déjà visitées ce jour (éviter les nouvelles requêtes)

**Message équipe** : *"On ne recharge pas l'appli — on reste sur ce qu'on a ouvert."*

---

### 🔋 Scénario 5 — Panne de courant au stand

**Symptôme** : Tout s'éteint.

**Action** :
1. Passer sur les **power banks** pour les téléphones immédiatement
2. Utiliser le formulaire papier pour capturer les leads
3. Contacter l'organisation du salon pour restaurer l'alimentation
4. Une fois le courant rétabli : rallumer dans l'ordre → laptop principal → écran → laptop backup
